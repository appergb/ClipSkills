# 11 · 多机位与采访

> 对应课程第12集（媒体文件前缀14，注意"文件前缀=集号+2"）。原则：**拿到多机位素材第一件事是合板对齐，不是剪**；剪的本质是"导播切流——镜头永远对准正在说话的人"。

## 一、什么时候用多机位

| 场景信号 | 判断 | 决策 |
|:---|:---|:---|
| 同场景对话/采访/双人口播，多台机器同时录 | 用多机位避免重拍穿帮 | 走本分册流程 |
| 综艺、采访、电影常见的"切对话+反应" | 单机位接不住反应表情 | 多机位导播式切换 |
| 单机位、无第二视角素材 | 不适用 | 回 `01`/`02` 常规流程 |

整体三步骨架：**①合板对齐 → ②单机位粗剪保内容一致 → ③多机位导播式切换 → 最后包装调色**。

## 二、Step 1：合板对齐（最关键，不对齐后面全废）

各机位录制起止时间不可能完全一致（早晚几秒）。不对齐 = 声画不同步（A 机刚开口、B 机里嘴已闭）。**先对齐再谈剪**。

| 对齐法 | 适用 | 取舍 |
|:---|:---|:---|
| **音频波形对齐（推荐默认）** | 各机位都录到了同一声音 | 电脑检测"同时发出的同样声音"对齐；免费、准确、简单 |
| **时码器（Timecode）** | 高精度、现场有设备 | 现场给所有机位打统一时间码、后期自动对；贵 / 不万能 / 偏专业 |
| **剪映自动** | 普通场景图快 | 综合画面 + 元数据自动对，打板声明显时很准 |
| **手动观察波形峰值** | 自动失败时的保底 | 对齐各机位波形的同一个峰值（如拍手/打板） |

> Agent 决策：普通场景 → 音频波形 / 剪映自动；要高精度且现场有设备 → 时码器；自动对失败 → 手动看波形峰值。**长素材务必开代理模式（Proxy）** 保流畅。输出 `{对齐方式, 是否开代理}`。

## 三、Step 2：单机位粗剪（保内容一致）

先把"说了什么、留哪些内容"定下来——按 `02-音频驱动剪辑流程` 的字幕切点逻辑，确定保留的对话段与顺序。**这一步只管内容，不管切哪个机位**。产物 = 一条内容正确、时长锁定的主轴。

## 四、Step 3：导播式切换（核心操作）

把对齐好的多机位片段当"导播台"：左 = 多机位预览器（同时预览各机位）、右 = 时间线预览，右上角按机位数切视图。

切换逻辑（重写为 Agent 决策）：

```
遍历对话时间轴的每个"换镜点" →
  这一刻谁在说话？ → 镜头对准正在说话的人
  有人被提到/被调侃的精彩处？ → 切到对方机位捕捉反应表情
  输出 (切点时间戳, 目标机位)
```

| 依据 | 规则 |
|:---|:---|
| **谁在说话** | 默认对准当前说话人（说话人切换处即换镜点） |
| **内容重要性** | 一人讲到精彩/爆点时，切对方机位捕反应，比一直怼说话人更有戏 |
| **切错机位** | 不慌、不重切刀——选中该片段、重选正确机位即可 |

> 人类落地：不选中多机位片段→播放头移到目标人开口那一帧→鼠标进左侧多机位窗口（变剃刀）→点目标机位，时间线自动切一刀+切机位。Agent 只产出"切点时间戳 + 对应机位"清单，按钮交 editor。

## 五、Step 4：包装

| 包装件 | 何时上 | 要点 |
|:---|:---|:---|
| **人名条 / 角标** | 多人对话、采访需标明身份 | 文字模板搜"人名条"→改名→放画面**左下角且在安全框内**，模板自带动画 |
| **金句切片** | 出现传播力强的金句 | 单独提取该段 + 文字包装，做成短视频利于传播 |

最后再整体调色（见 `06-调色`），多机位多片段尤其要做色彩克隆统一调性。

> 黑场红线：导播切换若出现未覆盖的硬切/内容空缺，**不自动删主轴**，标"待裁"交回用户。

## 六、产物（交给后续步骤）

```
multicam = {
  align: { method:"音频波形/时码器/剪映自动/手动峰值", proxy:true },
  cuts: [ { t:00:01:23, cam:"A", reason:"对准说话人" },
          { t:00:01:31, cam:"B", reason:"捕捉反应" }, ... ],
  packaging: { name_tags:[{cam/人名, 位置:"左下安全框"}], golden_clips:[{t_in,t_out}] }
}
```

## 给"答疑模式"的速答

- **"多机位素材到手第一步做什么？"** → 合板对齐，不是开剪。普通场景用音频波形对齐，长素材先开代理。
- **"切镜头依据什么？"** → 镜头对准正在说话的人；讲到精彩处切对方机位拍反应。
- **"切错机位要重切吗？"** → 不用，选中那段重选正确机位即可。
- **"多人采访画面要标什么？"** → 加人名条（左下角、安全框内），有金句再单独切片做短视频。

## 落地说明（交给 editor-* 技能）

- **Palmier Pro**：是否原生支持多机位（multicam）/导播切换能力不确定——**先 ToolSearch/查官方文档**确认其 MCP 工具，再决定走原生多机位还是退化为"多轨手动切"。
- **代码视频（FFmpeg · Remotion）**：FFmpeg 按音频对齐较繁——需音频互相关（audio cross-correlation，可用 praat 或 python+scipy 求偏移）再 `-itsoffset` 对齐；导播切换本质是按时间段在多路视频间选流（`trim`/`concat`/`overlay` 拼）。**多机位剪辑交 NLE 通常更高效**。
- **剪映 / 其他 NLE**：剪映用"新建多机位片段 + 导播台"；达芬奇、Premiere 均有原生 multicam。本内核只产出 `{对齐方式, 切点时间戳与机位, 人名条/金句清单}`，具体按钮交对应 editor 技能。
