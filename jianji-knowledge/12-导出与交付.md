# 12 · 导出与交付

> 对应课程第5集（媒体文件前缀07，最终交付-导出管理与发布规范；注意"文件前缀=集号+2"），整合第14集 demo/export 分离与全流程的"代理/渲染备份"性能管理。原则：**导出参数跟随素材、不盲目拉高；管理靠规范命名 + 文件夹结构，不靠记忆**。

## 一、为什么导出是"最后一道坎"

导出是流程真正最后一关，参数错会**糊 / 文件过大 / 被平台二次压缩压得更惨**。Agent 的职责不是"点导出"，而是先算出**一组匹配的导出设置**，再交给 editor 落地。核心立场：分辨率/帧率跟随素材上限，码率选"推荐"不拉满，编码器默认 H.264。

## 二、三大画质参数（决策依据）

| 参数 | 规则 | Agent 怎么定 |
|:---|:---|:---|
| **分辨率（Resolution）** | 1080P=横 1920×1080 / 竖 1080×1920；4K UHD=横 3840×2160 / 竖 2160×3840 | **跟随素材上限**。1080P 素材强拉 4K 不会更清晰，只徒增体积——取素材原生分辨率 |
| **帧率（Frame Rate）** | 24 电影感 / 25 中国广电 / 29.97 NTSC / 30 标准 / 50 PAL / 60 运动 | **默认与草稿/素材一致**，不为"更流畅"盲目升帧 |
| **码率（Bit Rate）** | 信息密度，越高细节越多但文件越大 | **不是越高越好**：人眼看不出 + 平台二次压缩会压回去。选"推荐"或按平台×分辨率×帧率码率参考表 |

> 红线：**分辨率与帧率永远跟随素材**，AI 超清/补帧只是补救，不取代前期拍摄（见第六节）。

## 三、封装格式 / 编码器 / 音频（对照表）

| 维度 | 选项 | 默认决策 |
|:---|:---|:---|
| **封装格式** | MP4 / MOV | 互联网平台皆可，默认 MP4 |
| **编码器** | H.264（兼容最好、稳妥默认） vs HEVC/H.265（同画质体积约小一半，但导出慢、不支持老设备） | **默认 H.264**；仅对体积有特别要求或电脑很好才用 H.265 |
| **音频编码** | AAC 192kbps=基础档（纯语音）/ 256kbps（音乐类）/ 320kbps（高要求）/ PCM 无损（专业、不直接发布） | 短视频：纯语音 192/256，音乐 256/320 稳妥 |
| **采样率（Sample Rate）** | 48000Hz | 固定 48000，再高人耳无差 |
| **色彩空间** | Rec.709 | 网络交付默认 Rec.709 |

> ⚠️ 订正：课程笔记表格曾误写"AAC 128"，以字幕一手口播为准，基础档应为 **192kbps**。

## 四、交付清单（Agent 的导出决策顺序）

```
选封面 → 编辑封面 → 规范命名 → 设导出位置(export) → 分辨率 → 帧率 → 码率
→ 封装格式 → 编码器 → 音频质量 → 采样率 → 查 VIP → 导出
```

输出格式（Agent 产物，一组导出设置）：

> `{平台 → 画幅 → 分辨率/帧率/码率/编码器/音频参数} + 命名 + 落盘路径(export/)`

## 五、封面与命名规范

- **封面（Cover）**：默认第一帧常是黑场 → 点"前往选择"拖播放头选**有张力且简洁**的代表帧；可"编辑封面"用模板/自加字/上传自定义。
- **文件命名规范**：`日期+片名+分辨率+版本+平台`，示例 `20260110_混剪教学_4K_成片1_抖音`。**杜绝"最终版/最终版2/再也不改"**这类不可追溯命名。

## 六、AI 超清/补帧与 VIP 的边界（红线）

- AI 超清画质 / 智能补帧基于算法**补缺失像素或帧**：放大露馅、出伪影、显油腻。**仅作单段素材补救**，画质上限永远取决于原始素材分辨率与帧率，不取代前期拍摄。
- **VIP 功能**：导出界面会显示用了几个 VIP 功能；未开会员可能**无法导出含 VIP 功能的画面**——导出前必查，免得卡在最后一步。

## 七、平台画幅/分辨率速查（并入）

| 平台 | 画幅 | 分辨率 |
|:---|:---|:---|
| YouTube | 16:9 | 1920×1080 |
| TikTok / Reels / 抖音 | 9:16 | 1080×1920 |
| Instagram Feed | 1:1 | 1080×1080 |
| X | 16:9 或 1:1 | 1920×1080 / 1080×1080 |

## 八、工程文件管理（管理思维）

- **项目开始即建总文件夹** → 子文件夹按类别：`01素材 / 02音乐 / 03成片` + `音效 / 字幕 / demo / export`。
- 整文件夹拖入编辑器**自动按层级排序**；保证**可移动、素材不离线**（换机打开顺路径自动找回）。
- **demo（粗剪小样）/ export（最终成片）分离导出**，同项目所有文件集中总文件夹，便于拷盘归档。

## 九、性能与渲染备份（整合自全流程）

- **代理模式（Proxy）**：编辑器生成低分辨率代理供编辑，**导出时自动换回原始高清，不影响成片画质**。预览卡顿先**降预览画质**，仍卡再开代理。
- **耗算力操作（抠像/跟踪/补帧）一律先粗剪定稿后做**，做完**先导出备份**——尤其跟踪完导出成视频（不用复合片段打包，以防卡死），再继续后续剪辑。

## 给"答疑模式"的速答

- **"导出参数怎么填？"** → 分辨率/帧率跟素材，码率选"推荐"，封装 MP4、编码 H.264，音频 AAC 纯语音 192/音乐 320、采样 48000；导出前查 VIP。
- **"导 4K 会更清晰吗？"** → 不会。素材是 1080P，强拉 4K 只增体积不增清晰；AI 超清也只是补救，上限取决于原始素材。
- **"文件怎么命名/归档？"** → `日期_片名_分辨率_版本_平台`，demo 和 export 分开放总文件夹子目录，整盘可拷走。
- **"剪辑/导出太卡？"** → 先降预览画质再开代理；抠像跟踪等先粗剪后做，做完先导出备份。

## 落地说明（交给 editor-* 技能）

- **Palmier Pro**：导出 MP4（H.264/H.265/ProRes）与 NLE XML（交达芬奇/Premiere）。具体导出接口、可选参数与封面/命名能力 → 去 `editor-palmier-pro`（能力不确定先查其 MCP 工具或官方文档再用）。
- **代码视频（FFmpeg · Remotion）**：转码 `libx264 -crf 18`（质量）或 `-b:v` 指定码率，`-c:a aac -b:a 192k/256k/320k`，`-pix_fmt yuv420p`（兼容），H.265 用 `libx265`；画幅重构（横转竖/方）`crop` 中心裁切后必须 `scale` 到偶数尺寸（`scale=1080:1920` 或 `-2`）并加 `setsar=1`（libx264+yuv420p 要求宽高均偶数）。完整命令与参数表 → 去 `editor-code-video`。
- **剪映 · 其他 NLE**：导出弹窗逐项填（分辨率/帧率/码率/格式/编码/音频/封面/VIP 检查）→ 去 `editor-other-nle`；不确定某项菜单是否存在，先查官方文档再用。
- 黑场红线：未覆盖的硬切/空缺由前序分册处理，本册不自动删主轨内容，遇缺标注交回用户。
