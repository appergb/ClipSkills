# 07 · 关键帧与运动控制

> 对应课程第6集（媒体文件前缀08，注意"文件前缀=集号+2"）。原则：**关键帧给参数注入时间维度，多参数必须对齐、多曲线必须同步**——动作脱节几乎都源于这两条被破坏。

## 一、关键帧的本质（先建立模型）

- 关键帧（Keyframe）= 给某个静态参数设两个不同值的"时间锚点"（起点 + 终点），软件按时间顺序自动计算中间过程 = 补间动画（Tween）。
- 可打帧的参数：位置（Position）/ 缩放（Scale）/ 旋转（Rotation）/ 不透明度（Opacity）。每个参数后有菱形按钮（◆）。
- **判断状态的唯一可靠依据**：空心菱形 ◇ = 该参数无关键帧；实心菱形 ◆ = 已有关键帧。多参数动画时，靠它确认"到底是哪个参数被打了帧"，别靠记忆。

## 二、何时打关键帧 → 选哪种缓动（核心决策）

| 情况 | 依据 | 决定 | 输出给 editor |
|:---|:---|:---|:---|
| 参数需随时间变化（推拉、移动、淡入淡出、旋转） | 静态画面不够，需要运动 | 起点 + 终点各打一帧 | `{参数, 起止帧, 起止值}` |
| 默认直线运动显得机械、不自然 | 现实物体有加减速 | 套"缓入缓出（Easing）"曲线预设 | `{曲线类型: 二次方缓入缓出}` |
| 需要"砸下来"的冲击感 | 末段要快、突然停 | 用强缓出 / 自定义贝塞尔 | `{曲线类型: easeOut/自定义贝塞尔}` |

- 参数调整四方式（精度递增，按需选）：拖滑块（快/粗）→ 点上下箭头（每次 ±1/精）→ 直接输入数值（最精）→ 播放器内拖边框缩放、拖画面移位（最直观）。
- **片段参数相互独立**：同一素材切成两段，两段参数互不关联。要批量改，先**全选**再到参数面板改。

## 三、经典案例："后期拉近"必须双参数

- 目标：边放大边推向画面中央（模拟镜头推近）。
- **只打缩放帧 = 原地放大**，主体不会移到中心 → 必须**同时给位置（X/Y）打帧**，缩放与位置组合才是"拉近"。
- 这是"多参数关键帧"最常见的需求，本例共 3 条曲线：缩放 1 条 + 位置 X / Y 各 1 条。

## 四、对齐与同步（两条铁律，违反即脱节）

1. **关键帧位置对齐**：给新参数打帧时，播放头要**对齐已有关键帧的帧位置**，保证所有参数同时开始、同时结束。错位的表现 = "先放大再移动" 或 "先移动后放大"。
   - 自检：用 `Command + 滚轮`（Mac）/ `Ctrl + 滚轮`（Win）放大时间线，逐帧核对各参数菱形是否对齐。
2. **缓动曲线同步**：只给缩放套缓动、而 X/Y 仍是线性 → 动作脱节（先放大才移动）。**所有相关参数套同一套缓动预设**。
   - 关键帧曲线工具（剪映称"关键帧变速曲线"）：右键片段 或 `Option + K`（Mac）/ `Alt + K`（Win）。
   - 曲线读法：横轴 = 播放时间，纵轴 = 参数值；默认直线 = 匀速；套"二次方缓入缓出"→ S 型曲线 = 自然加减速。

## 五、Agent 输出规范

判断完成后，把每个被打帧的参数整理成结构化指令交给 editor：

```
{ 参数: position.x, 关键帧: [{帧:0, 值:0}, {帧:30, 值:-200}], 曲线: 缓入缓出 }
{ 参数: scale,      关键帧: [{帧:0, 值:100},{帧:30, 值:150}], 曲线: 缓入缓出 }
```

- 帧位置务必跨参数一致（同 0、同 30）；曲线类型务必跨参数一致。
- 缺帧、空缺区段不自动补救或删主轨内容，按黑场红线**标注交回用户**确认。

## 给"答疑模式"的速答

- **"拉近怎么只放大没往中间走？"** → 你只打了缩放帧，必须**同时给位置打帧**，缩放 + 位置组合才是拉近。
- **"动画为什么先放大再移动、很别扭？"** → 关键帧没对齐或曲线没同步；放大时间线对齐帧位置，并给所有参数套同一缓动预设。
- **"怎么让运动不那么机械？"** → 默认是匀速线性，套"缓入缓出"曲线预设即可模拟自然加减速。
- **"怎么知道哪个参数被打帧了？"** → 看参数后的菱形：实心 ◆ = 有帧，空心 ◇ = 无帧。

## 落地说明（交给 editor-* 技能）

- **Palmier Pro（`editor-palmier-pro`）**：经 MCP 调 `transform`（含 scale / position）、`opacity`。先用 **ToolSearch** 查出真实工具名与参数 schema 再调，不确定的能力一律先查官方文档，勿臆造。
- **代码视频（`editor-code-video`，Remotion / FFmpeg）**：Remotion 用 `useCurrentFrame()` + `interpolate(frame,[in],[out],{extrapolateLeft:'clamp',extrapolateRight:'clamp'})` 驱动；**只动 `transform`/`opacity`，绝不动 `width`/`height`/`top`/`left`**；缓入缓出用 `spring()` 或贝塞尔。FFmpeg 对关键帧动画支持弱，仅 `zoompan` 可做缓慢推拉，复杂运动改走 Remotion（该技能再指向 `remotion-video-creation`）。
- **剪映 / 其他 NLE（`editor-other-nle`）**：剪映打帧按钮 = 菱形 ◆，曲线工具 = `Option + K`（Mac）/ `Alt + K`（Win）；其他 NLE 的具体按钮路径先查官方文档再操作。
