# 03 · 素材 A/B 分类与替换

> 对应课程第2集（A/B-roll 协同）与第3集。核心动作：**按字幕语义，把内容匹配的 B-roll 覆盖到 A-roll 对应时间位置**。
> 这就是用户说的"口播剪辑里把素材替换到音频对应位置"的操作。

## 一、A-roll / B-roll 定义

| 概念 | 含义 | 轨道位置 |
|:---|:---|:---|
| **A-roll** | 主线信息（口播讲解、旁白、采访同期声） | 主轨 |
| **B-roll** | 叠加的补充画面（空镜、演示、动效、相关镜头） | 上层叠加轨 |

A-roll 决定"说什么"，B-roll 决定"看什么"，相辅相成。

## 二、引导用户做 A/B 分类（反问模式）

动手前确认素材是否已分类。若没分，给用户清晰的分类标准：

> **请把素材分两堆：**
> - **A 类（主线）**：承载主要信息的素材——口播/讲解/采访同期、旁白对应的主画面。通常一条主线贯穿。
> - **B 类（补充）**：用来"配合 A 在讲什么"的画面——空镜、产品/操作演示、氛围镜头、同一对象不同景别/角度的镜头组。
>
> 命名建议：总文件夹用`日期-内容`；子文件夹`01素材/02音乐/03成片/demo/export`；视频保留相机原始命名避免格式错误。完整的命名规范与工程文件管理见 `12-导出与交付.md`。

> 经验：A/B 还可继续按主题分（如本片的"主题甲/主题乙"两类 B-roll），匹配时更快。

## 三、内容匹配替换（Agent 版"筛子法"）

课程的"双时间线筛子法"是人肉扫素材；Agent 版用**语义检索**替代：

```
for 每个 audio_timeline 段 (text, t_in, t_out):
    keywords ← 从 text 提取关键词/实体/动作
    候选 ← 在 B 类素材库里按 keywords 检索（文件名/转写/标签/画面描述）
    选 1~N 段最匹配的 B-roll
    覆盖到上层轨道的 [t_in, t_out]
输出：匹配映射表 [{段, 选用素材, 起止, 备注}]
```

落地（覆盖到时间线）由 editor 技能执行；本内核只产出**匹配映射表**并遵守下面五条。

## 四、贴 B-roll 五注意（课程铁律）

1. **对齐口播节奏**：B-roll 时长要盖住对应那句话；太短会"话没说完画面就切了"（漏字出戏）。不够长就换更长的素材或在切点收尾。
2. **成组添加**：同一内容、不同景别/角度的几段组成镜头组，比单镜更耐看。
3. **遮蔽硬切**：A-roll 上无法被 B-roll 覆盖的硬切——① 放大该段素材当作"第二机位"遮蔽；② 用与主题相关的"万能 B-roll/空镜"填空。
4. **不要重复**：同一条片子里别出现一模一样的 B-roll，显得不讲究。
5. **B-roll 静音**：B-roll 通常是无声镜头；统一放同一条轨道，整轨静音。

## 五、与黑场红线的关系

- 凡是"找不到合适 B-roll 覆盖、又不能放大遮蔽"的硬切/空缺 → 不要 Agent 自己删主轨凑接，**标注为黑场，交回用户**手动裁切。
- 大批量替换前先输出匹配映射表给用户过目，再落地。

## 六、给"答疑模式"的速答

- **"B-roll 怎么选/怎么贴？"** → 按字幕这一句在讲什么去库里找对应画面，盖住整句时长，成组、不重复、静音，硬切处用放大或空镜遮。
- **"画面老是切太快漏字怎么办？"** → B-roll 太短，换更长素材或让它盖到这句话说完再切。
